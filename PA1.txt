1. Color Resource(res/values/colors.xml)
Purpose: To define color values used in the app. Instead of hardcoding
color values in your layout files, you can reference them as resource IDs.
This makes it easy to change the app's color scheme across different
screens.
Example (res/values/colors.xml):

<resources>
<color name="primaryColor">#3F51B5</color> <!-- Blue color -->
<color name="accentColor">#FF4081</color> <!-- Pink color -->
<color name="backgroundColor">#FFFFFF</color> <!-- White
background --> </resources>

You can reference these colors in your layout or styles like this:
android:background="@color/primaryColor"
android:textColor="@color/accentColor"

Android Practical

23-122

2. Theme Resource (res/values/styles.xml):
Purpose: Themes define a set of styles that can be applied to an
entire application or specific activities.
Themes are often used to define the overall appearance, such as
colors, font styles, and button styles.

3. String (res/values/strings.xml) :
Purpose: To define text values used throughout the app.
By defining strings in a separate XML file, you can easily localize your
app into different languages. It helps in making your app scalable and
adaptable.

Android Practical

4. Drawable (res/drawable/) :
Purpose: Drawables are graphic resources like images (PNG, JPEG,
etc.),
XML shapes, and vector images.
They can be used as backgrounds, icons, buttons, and other visual
elements.
Example:
You can define a drawable like a custom shape or background in
XML.
Drag and drop the image in drawable folder and click ok change the
background layout color or images.

5. Dimension (res/values/dimens.xml):
Purpose: To define fixed sizes (like padding, margin, or font size) that
can be referenced throughout the app.
This allows the app to be more flexible, as changes to the UI layout (like
padding or margins) can be done in one place, and the app will scale
accordingly.

Aim: Programming activities
Introduction: In Android development, an Activity is a single screen with a user
interface. Every Android app is made up of one or more activities, each representing a
different screen or interaction point. A Fragment is a modular section of an activity
that has its own lifecycle, receives its own input events, and can be added or removed
while the activity is running.

Activity Lifecycle:

Android Practical

23-122

Activity Life Cycle:
Every Android Activity goes through a series of states from creation to
destruction. Android provides callback methods that are triggered at each
stage:


onCreate() - Called when the activity is first created. Used to
initialize UI components and set the content view.



onStart() - Called when the activity becomes visible to the user.



onResume() - Called when the activity starts interacting with the user.
The activity is now in the foreground.



onPause() - Called when the system is about to resume another
activity. Used to save unsaved changes.



onStop() - Called when the activity is no longer visible to the user.



onRestart() - Called when the activity is restarting after being
stopped.



onDestroy() - Called before the activity is destroyed. Used to release
resources.

Activity Methods:
Key methods used in an Activity include:


setContentView() - Sets the layout XML file to be displayed.



findViewById() - Finds a UI element by its ID.



startActivity() - Launches a new activity using an Intent.



finish() - Closes the current activity and returns to the previous one.



getIntent() - Retrieves the Intent that started the activity, used to
receive passed data.

Multiple Activities:

Android Practical

23-122

An Android application can contain multiple activities. Navigation between
activities is handled using Intents. An Explicit Intent is used to start a
specific activity within the same app by directly specifying the target class.
Data can be passed between activities using putExtra() and retrieved using
getExtra().
Fragment Life Cycle:
A Fragment has its own lifecycle which is closely tied to its host Activity.
The key lifecycle methods of a Fragment are:


onAttach() - Called when the fragment is attached to its host activity.



onCreate() - Called to initialize the fragment.



onCreateView() - Called to inflate and return the fragment's layout.



onActivityCreated() - Called when the host activity's onCreate() has
completed.



onStart() - Called when the fragment becomes visible.



onResume() - Called when the fragment starts interacting with the
user.



onPause() - Called when the fragment is no longer interacting with
the user.



onStop() - Called when the fragment is no longer visible.



onDestroyView() - Called when the fragment's view is being
removed.



onDetach() - Called when the fragment is detached from its activity.

Multiple Fragments:
Multiple fragments can be used within a single activity to create a flexible
and dynamic UI. Fragments are managed using the FragmentManager and
FragmentTransaction classes. Common operations include:


add() - Adds a fragment to the container

Android Practical

23-122



replace() - Replaces an existing fragment with a new one.



remove() - Removes a fragment from the container.



addToBackStack() – Adds the transaction to the back stack so the
user can navigate back to the previous fragment by pressing the back
button.



commit() - Commits the transaction to apply all changes.

Practical 4
AIM: Programs related to different Layouts: Coordinate,
Linear, Relative, Table, Absolute, Frame, List View, Grid View.
1 Linear Layout
Definition:
Linear Layout is a layout that arranges views in a single row or a single
column.
Working:
It uses the orientation attribute (vertical or horizontal) to place
elements one after another in a straight line.

2 Relative Layout
Definition:
Relative Layout is a layout that positions views relative to each other or
relative to the parent layout.
Working:
It uses positioning rules such as below, above, alignParentTop,
and centerInParent to arrange views.

3 Frame Layout
Definition:
Frame Layout is a layout that displays one view or stacks multiple views
on top of each other.
Working:
It shows the view placed inside it. If multiple views are added, they
overlap. If no view is added, nothing is displayed.

Android Practical

23-122

4 Table Layout

Defination:
TableLayout is a layout that organizes views into rows and columns like
a table.
Working:
It uses TableRow to create rows, and views placed inside each row act
as columns.

5 Absolute Layout (Deprecated)
Definition:
Absolute Layout is a layout that places views using fixed X and Y
coordinates.
Working:
It positions elements using exact screen coordinates. It is not responsive
and is not recommended for modern Android development.

6 ListView
Definition:
ListView is a view that displays a scrollable vertical list of items.
Working:
It uses an Adapter to connect data (like Apple, Banana, Mango) to the
list and display each item.

7 GridView
Definition:
GridView is a view that displays items in rows and columns in grid
format.
Working:
It uses the numColumns property and an Adapter to arrange items in a
grid style.
